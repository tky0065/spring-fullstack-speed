Questions fréquentes

Comment installer Spring-Fullstack-Speed?

Vous pouvez installer Spring-Fullstack-Speed globalement via npm:

npm install -g @enokdev/spring-fullstack-speed

Ou l'utiliser directement avec npx:

npx @enokdev/spring-fullstack-speed

Quels sont les prérequis pour utiliser SFS?

Pour utiliser Spring-Fullstack-Speed, vous aurez besoin de:

  • Node.js v18.0.0 ou supérieure
  • npm v8.0.0 ou supérieure
  • Java JDK 17 ou supérieure
  • Maven ou Gradle (pour la compilation des projets Spring Boot)

Comment mettre à jour SFS vers la dernière version?

Pour mettre à jour Spring-Fullstack-Speed vers la dernière version, utilisez la commande npm update:

npm update -g @enokdev/spring-fullstack-speed

Comment générer une nouvelle entité avec des relations?

Pour générer une entité avec des relations, utilisez la commande sfs entity avec l'option --relations:

sfs entity --name=Product --fields=name:string,price:double --relations=many-to-one:Category:category

Les types de relations disponibles sont: one-to-one, one-to-many, many-to-one et many-to-many.

Comment personnaliser les templates générés?

Vous pouvez personnaliser les templates en créant un dossier ~/.sfs/templates/ et en y copiant les templates que vous souhaitez modifier. SFS utilisera automatiquement vos templates personnalisés.

Par exemple, pour personnaliser le template d'entité:

  1. Créez le dossier: mkdir -p ~/.sfs/templates/entity
  2. Copiez le template original: cp node_modules/@enokdev/spring-fullstack-speed/generators/entity/templates/Entity.java.ejs ~/.sfs/templates/entity/
  3. Modifiez le template selon vos besoins

Mon application ne se lance pas, comment résoudre ce problème?

Si votre application générée ne se lance pas, voici quelques étapes de dépannage:

  1. Vérifiez les logs d'erreur dans target/spring.log
  2. Assurez-vous que votre base de données est en cours d'exécution et accessible
  3. Vérifiez que les informations de connexion à la base de données dans application.yml sont correctes
  4. Utilisez la commande sfs doctor pour diagnostiquer les problèmes courants

Si le problème persiste, n'hésitez pas à ouvrir une issue sur GitHub ou à nous contacter via le formulaire ci-dessous.

Comment contribuer au projet Spring-Fullstack-Speed?

Nous sommes ravis que vous souhaitiez contribuer! Voici les étapes:

  1. Forkez le dépôt sur GitHub
  2. Clonez votre fork localement
  3. Créez une branche pour votre contribution
  4. Faites vos modifications
  5. Exécutez les tests
  6. Soumettez une Pull Request

Pour plus de détails, consultez notre guide de contribution.
